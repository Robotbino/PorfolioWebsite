C5

Ref-counted scroll lock — one owner for overflowY

✓ shipped · 3 commits

Was: two copy-pasted implementations of the root overflowY lock (mobile menu + spotlight) shared one DOM property with no owner, so whoever released first clobbered the other. The spotlight also lacked the iOS touchmove workaround.

Now: ScrollLockService owns the property. acquire() ref-counts and returns an idempotent release handle; the lock engages on the first acquire and releases only when the last holder lets go. Both callers are adapters; the iOS workaround is written once.

  • Clobber bug impossible
  • iOS touchmove workaround written once (spotlight gains it)
  • Idempotent handle — double-release / destroy-then-close safe

C4

Motion-capability seam + hidden-tab pause

✓ shipped · 9 commits

Was: every visual system privately re-implemented "may I animate" — reduced-motion read ×7 (app, site-nav ×2, constellation, about, work, certifications) and pointer ×2 (the original ×3 miscounted work's min-width query as pointer), each frozen at init. And while browsers park rAF in hidden tabs anyway, the frame pulse handed the whole hidden gap to the first tick back as dt, lurching every smoothing consumer on tab-return.

Now: MotionSettingsService exposes reducedMotion / finePointer / coarsePointer as live signals (matchMedia change listeners — the ThemeService pattern). Fine and coarse stay two independent signals: they're non-complementary (a hybrid touch-laptop matches neither), so each consumer keeps its exact question and behaviour is unchanged. FramePulse cancels rAF on visibilitychange→hidden and re-arms with lastTime reset, so the first visible tick reports dt = 0 — the onTick interface untouched.

  • 9 private matchMedia reads → 1 seam; zero left in TS
  • Live preference flips propagate (consumers opt in later)
  • Tab-return dt spike killed — no smoothing lurch
  • Renderers stay dumb adapters

C9

One in-viewport seam — the IntersectionObservers

✓ shipped · 6 commits · the last candidate

Was: "element entered the viewport" was re-implemented five times with different thresholds, lifecycles (one-shot vs repeating), and an inconsistent zone story — about deliberately ran outside Angular, certs ran inside wrapping one write, the rest never thought about it. C3 had knowingly added the fifth (the #projects highlight) to keep that commit tight.

Now: InViewportService — observe(target, options, onChange) → release, the ScrollLockService idiom. Observers are shared by serialized option set (root is always the viewport, so rootMargin + threshold identify a reusable instance watching many targets): nav #projects + the about-glow icons ride one middle-line observer; the shell reveal + every appScrollReveal share the 0.15 one — five observers → three. The callback runs outside Angular; only certs' armed.set re-enters via zone.run. The directive keeps its template API and delegates; one-shot consumers just call release() in their callback.

  • 5 private observers → 1 owner, 3 shared instances
  • Zero raw new IntersectionObserver outside the seam
  • About glow shed its own NgZone; zone story consistent
  • Release: idempotent, unobserve, disconnect-when-empty

5 · The deep core now

ScrollLoopService + scroll-loop.math — the Cycle behind a 6-member interface now; DOM I/O stays in the shell, math is pure and spec'd. Deepened with activeDestination (C3).
destinations.ts — the single registry the shell, nav, and constellation order consume. New (C2); spec-guarded against drift.
ScrollLockService — one ref-counted owner of the root scroll lock + iOS workaround, behind an acquire() → handle interface. New (C5).
motion.math — the frame-rate-independent smoothing, one tested copy. New (C8).
FramePulseService — one rAF, out of zone, ref-counted, one-method interface; pauses on hidden tabs and resets its clock on return so the first tick back reports dt = 0. Deepened (C4) behind the unchanged interface.
MotionSettingsService — the one "may I animate, and how does the visitor point?" answer: three live readonly signals every renderer consumes. New (C4); zero motion/pointer matchMedia left elsewhere.
InViewportService — the one "is this element in the viewport?" owner behind observe() → release, sharing observers by config and running callbacks out of zone. New (C9); absorbed all five raw observers.
MorphDriver + constellation-morph + figures — morph physics/interpolation as pure, spec-covered modules; the component is a thin SVG adapter.
certifications.math — tilt, mask, FLIP arithmetic, spec'd; smoothingK moved out to motion.math (C8).
theme.decision.ts + aurora-palette.ts — one theme decision rule and one theme→palette mapping, both pure and spec'd. New (C7 + C6); closed the last system-map leak.
Aurora's @Input seam — the only pixel renderer, 4 inputs hiding 300 lines + GLSL; palette now theme-owned and memoized off the frame tick (C6).
