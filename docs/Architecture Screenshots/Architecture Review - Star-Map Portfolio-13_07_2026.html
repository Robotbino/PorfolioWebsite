C2

✓ shipped · 4 commits

C5

Ref-counted scroll lock — one owner for overflowY

✓ shipped · 3 commits

Was: two copy-pasted implementations of the root overflowY lock (mobile menu + spotlight) shared one DOM property with no owner, so whoever released first clobbered the other. The spotlight also lacked the iOS touchmove workaround.

Now: ScrollLockService owns the property. acquire() ref-counts and returns an idempotent release handle; the lock engages on the first acquire and releases only when the last holder lets go. Both callers are adapters; the iOS workaround is written once.

  • Clobber bug impossible
  • iOS touchmove workaround written once (spotlight gains it)
  • Idempotent handle — double-release / destroy-then-close safe

5 · The deep core now

ScrollLoopService + scroll-loop.math — the Cycle behind a 6-member interface now; DOM I/O stays in the shell, math is pure and spec'd. Deepened with activeDestination (C3).
destinations.ts — the single registry the shell, nav, and constellation order consume. New (C2); spec-guarded against drift.
ScrollLockService — one ref-counted owner of the root scroll lock + iOS workaround, behind an acquire() → handle interface. New (C5).
motion.math — the frame-rate-independent smoothing, one tested copy. New (C8).
FramePulseService — one rAF, out of zone, ref-counted, one-method interface. C4 will deepen it with a hidden-tab pause.
MorphDriver + constellation-morph + figures — morph physics/interpolation as pure, spec-covered modules; the component is a thin SVG adapter.
certifications.math — tilt, mask, FLIP arithmetic, spec'd; smoothingK moved out to motion.math (C8).
Aurora's @Input seam — the only pixel renderer, 4 inputs hiding 300 lines + GLSL. C6 fixes what feeds it, not the module.
